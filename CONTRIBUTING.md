# Contributing to Cumulocity Widget Library

Thank you for your interest in contributing to the Cumulocity Widget Library! We welcome contributions from the community to help make this project even better.

By contributing to this repository, you agree to license your contribution under the terms of the Apache License, Version 2.0, and agree to the terms in the `CONTRIBUTOR-LICENSE-AGREEMENT.md`.

## How to Contribute

### 1. Submit Issues
If you find a bug or have a feature request, please open an issue in the GitHub repository. Provide as much detail as possible, including:
- Steps to reproduce (for bugs).
- What you expected to happen vs what actually happened.
- Screenshots if applicable.
- Environment details (browser, Cumulocity version, etc.).

### 2. Pull Requests
We accept contributions via Pull Requests (PRs). To submit a PR:
1. Fork the repository and create your branch from `main`.
2. Ensure your code follows the repository's coding style (Angular/TypeScript guidelines, Prettier configuration).
3. Add or update unit tests as appropriate.
4. Ensure all unit tests pass (`npm run test`).
5. Build the project successfully (`npm run build`).
6. Document your changes in the PR description and update `README.md` if necessary.
7. Open the PR targeting the `main` branch.

## Code Style & Standards
- Keep code clean, readable, and well-commented where appropriate.
- Ensure all source files contain the copyright header:
  ```ts
  /*
   * Copyright (c) 2026 Cumulocity GmbH.
   *
   * SPDX-License-Identifier: Apache-2.0
   */
  ```
- Run formatting and build checks before submitting code.
